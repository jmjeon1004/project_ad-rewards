# Part I — 프로젝트 정의

## 1. 비전과 목적

### 1.1 해결하는 문제

아이브코리아는 수백 건의 광고를 동시 운영하며, 매일 다음과 같은 의사결정을 반복한다:

- **어떤 광고가 잘 되고 있는가?** — 성과 지표(CVR, CPA, 마진율)가 흩어진 데이터 소스에 산재
- **신규 광고의 품질을 사전에 평가할 수 있는가?** — 등록 시점 속성만으로 품질 등급을 판별하는 기준 부재
- **부진 광고를 얼마나 빨리 포착할 수 있는가?** — D+3 초기 실적만으로 부진 위험을 감지하는 조기 경보 체계 부재
- **근거 기반 판단을 내릴 수 있는가?** — 직관과 경험에 의존하는 운영, 데이터 근거 기반 표준화된 판단 프레임워크 필요

### 1.2 솔루션

Streamlit 기반 5페이지 멀티페이지 대시보드로, 4가지 핵심 역량을 하나의 인터페이스에 결합한다:

```
┌──────────────────────────────────────────────────────────┐
│                    광고운영 최적화 대시보드                  │
│                                                          │
│  ① 성과 분석        ② ML 품질 등급     ③ 부진 예측       │
│  (KPI, 히트맵,      (XGBoost M1:      (LightGBM M2:     │
│   사분면, 추이)      S/A/B/C/D)        decline_risk)     │
│                                                          │
│           ④ AI Agent 자연어 질의 (Gemini 2.5 Flash)       │
└──────────────────────────────────────────────────────────┘
```

### 1.3 기대 효과

| 영역           | Before             | After                                                        |
| -------------- | ------------------ | ------------------------------------------------------------ |
| 성과 모니터링  | 수동 엑셀 집계     | 실시간 대시보드 + 시간대/요일 패턴 자동 탐지                 |
| 신규 광고 평가 | 감에 의존          | M1 품질 등급(S~D) + 점수(0~100) 객관적 판별                 |
| 부진 감지      | 1~2주 후 사후 분석 | D+3 시점 조기 경보 (M2 부진확률)                             |
| 운영 액션      | 담당자별 기준 상이 | 배지 기반 표준화된 액션 제시 (매체확장/등급경계/즉시조치 등) |
| 애드혹 분석    | SQL 작성 필요      | AI Agent 자연어 질의                                         |

## 2. 대상 사용자와 시나리오

### 2.1 페르소나

| 역할                       | 주요 관심사                         | 사용 패턴                                                       |
| -------------------------- | ----------------------------------- | --------------------------------------------------------------- |
| **광고 운영 매니저** | 일일 성과 변동, 긴급/주의 광고 파악 | 매일 오전 P1 오버뷰 확인 → 이상 감지 시 P2~P4 드릴다운         |
| **미디어 바이어**    | 매체별 효율, 예산 배분 최적화       | P3 매체별 탐색 → 버블차트로 효율 비교 → P5 매체확장 액션      |
| **운영 의사결정자**  | 포트폴리오 전체 건전성, ML 인사이트 | P1 배너(정상/주의/긴급 비율) → ML 인사이트 탭 → AI Agent 질의 |

### 2.2 일상 워크플로우

```
[오전 루틴]
P1 오버뷰 → 운영 상태 배너 확인 (정상 80% 이상?)
         → KPI 카드 (CVR, 마진율 추이)
         → 히트맵에서 시간대/요일 이상 패턴 탐지
         → ML 인사이트에서 기회/위험 광고 확인

[이상 감지 시]
P1 Top3 카드 → "유형별 탐색 →" 클릭 → P2 진입
P2~P4 → 필터 조정 → 버블차트 사분면 분류 → 운영 알림 카드 확인
      → CVR 추이에서 트렌드 변화 확인

[액션 실행]
ML 인사이트 "상세" 버튼 → P5 광고 상세
P5 → 배지별 액션 카드 (매체확장/등급경계/즉시조치/우선검토)
   → 구체적 액션 실행 (결재 요청, 예산 확보, 일시 중단 등)

[애드혹 분석]
각 페이지 하단 AI Agent → "가입형 광고 중 CVR이 가장 낮은 매체는?" 등 자연어 질의
```

## 5. 데이터 아키텍처

### 5.1 원천 데이터 파일

모든 파일은 `data/` 디렉토리에 Parquet 형식으로 저장된다. 핵심 JOIN 키는 `ads_idx`이다.

| #  | 파일                                  | 역할                                              | 주요 컬럼                                                                                                                                                                                                                 |
| -- | ------------------------------------- | ------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1  | `hourly_report.parquet`             | **시간대별 광고 성과 fact table**           | `rpt_time_date`, `rpt_time_time`, `rpt_time_clk`(클릭), `rpt_time_turn`(전환), `rpt_time_acost`(광고비), `rpt_time_scost`(정산비), `rpt_time_earn`(수익), `rpt_time_cost`(비용), `ads_idx`, `mda_idx` |
| 2  | `ad_attr_map.parquet`               | 광고 속성 (유형, 카테고리, 보상 등)               | `ads_idx`, `ads_name`, `ads_type_label`, `category_name`, `ads_reward_price`, `reward_band`, `ads_rejoin_type`, `ads_save_way`, `ads_day_cap`, `ads_sdate`, `ads_edate`                             |
| 3  | `ive_ad_classification.parquet`     | 매체/액션 분류                                    | `ads_idx`, `final_media`, `final_action`                                                                                                                                                                            |
| 4  | `ad_outcome.parquet`                | 전환 결과 (CTIT 1순위 소스)                       | `ads_idx`, `avg_ctit`, `is_valid_click10`, `click_cnt`                                                                                                                                                            |
| 5  | `main_funnel.parquet`               | 원시 퍼널 이벤트 ("오늘" 날짜 결정용)             | `click_date`                                                                                                                                                                                                            |

| 10 | `user_daily_activity_clean.parquet` | 유저 일별 활동                                    | —                                                                                                                                                                                                                        |

### 5.2 분석 기간

### 5.3 "오늘" 날짜 기준

"오늘"은 `main_funnel.parquet`의 `click_date` 최댓값(`load_today_date()`)으로 결정한다. 실제 시스템 날짜가 아닌 데이터 기준 날짜를 사용하여 분석의 일관성을 보장한다.

### 5.4 전처리 파이프라인

```

                    ├── load_ad_attr_map()

   resource)
                         │

              build_base_table()           ← src/preprocessing.py
              ─────────────────
              1. hourly_report + ad_attr_map       (LEFT JOIN on ads_idx)
              2. + ive_ad_classification            (LEFT JOIN on ads_idx)
              3. + ad_outcome                       (LEFT JOIN on ads_idx)

              5. final_action → analysis_ads_type_label 매핑 (ACTION_TYPE_MAP)
              6. final_media → 한글 매체명 매핑 (MEDIA_NAME_MAP)
              7. category_name: "선택안함" → "기타"
              8. 결측 → "기타"
                         │

              ──────────────────

                         │
                         ▼
              score_all_ads()              ← src/model.py
              ────────────────

                         │

```

### 5.5 세션 초기화 (`ensure_data_loaded()`)

앱 최초 접근 시 `src/init.py`의 `ensure_data_loaded()`가 1회 실행되며, 결과를 `st.session_state`에 저장한다. 이후 페이지 전환 시에도 재로딩하지 않는다.

| `today`             | 데이터 기준 "오늘" 날짜                |
| `ad_summary`        | 광고별 1행 집계 + ML 스코어 병합       |
| `attr`              | 광고 속성 원본                         |
| `classification`    | 매체/액션 분류 원본                    |
| `ad_master`         | 광고 마스터 원본                       |
| `sched`             | 스케줄 원본                            |
| `ad_outcome_full`   | 전환 결과 원본                         |

### 5.6 캐싱 전략

## 6. 지표 체계

### 6.1 도메인 규칙

> **모든 비율 지표는 합계 기반으로 계산한다.** 행별 비율의 산술평균은 사용하지 않는다.

> **분모가 0일 때는 `None`(null)을 반환하고, UI에서는 `-`로 표시한다.**

### 6.2 핵심 KPI 정의

| 지표                             | 수식                                             | 함수                         | UI 포맷                                |
| -------------------------------- | ------------------------------------------------ | ---------------------------- | -------------------------------------- |
| **CVR** (전환율)           | `sum(turn) / sum(clk) × 100`                  | `calc_cvr()`               | `{n}%`                               |
| **CPA** (전환단가)         | `sum(acost) / sum(turn)`                       | `calc_cpa()`               | `{n:,}원`                            |
| **마진율**                 | `(sum(acost) - sum(earn)) / sum(acost) × 100` | `calc_margin_rate()`       | `{n}%`                               |
| **총 클릭수**              | `sum(clk)`                                     | inline                       | K/M 축약                               |
| **총 전환수**              | `sum(turn)`                                    | inline                       | K/M 축약                               |
| **마진액**                 | `sum(acost) - sum(earn)`                       | inline                       | `{n:,}원`                            |
| **평일 vs 주말 CVR 격차**  | `cvr_weekday - cvr_weekend`                    | inline                       | `{n}%p`                              |
| **TOP 10 의존도**          | `sum(전환수 상위10) / sum(전체 전환) × 100`   | `calc_dependency_ratio()`  | `{n}%`                               |

### 6.3 집계 레벨

| 함수                               | 집계 레벨 | 반환 형식                                                                          |
| ---------------------------------- | --------- | ---------------------------------------------------------------------------------- |
| `calc_all_kpis(df)`              | 전체 단일 | dict:`{clk, turn, acost, scost, earn, cvr, cpa, margin_rate}` |
| `calc_group_kpis(df, group_col)` | 그룹별    | DataFrame (그룹 컬럼 + KPI 컬럼)                                                   |
| `calc_daily_kpis(df)`            | 일자별    | DataFrame (`rpt_time_date` + KPI)                                                |
| `calc_hourly_kpis(df)`           | 시간대별  | DataFrame (`rpt_time_time` + KPI)                                                |

### 6.4 전기 비교 (Period-over-Period)

`calc_change_rate(current, previous)` → 단순 차이 (`current - previous`).

| 현재 기간         | 비교 기간       | 비교 방식                 |
| ----------------- | --------------- | ------------------------- |
| 최근 1일          | 전일 1일        | 차이                      |
| 최근 7일          | 직전 7일        | 차이                      |
| 사용자 지정 (N일) | 시작일 직전 N일 | 차이                      |
| 전체              | —              | 비교 없음 (변화량 미표시) |

# Part III — ML 모델

## 7. M1: 광고 품질 등급

### 7.1 목적

광고 등록 시점의 속성(보상가, 유형, 매체, 일캡 등) **만으로** 품질 등급을 예측한다. 운영자가 신규 광고의 잠재력을 사전에 파악하고, 자원 배분 우선순위를 결정하는 데 활용한다.

### 7.2 모델 사양

| 항목      | 내용                                              |
| --------- | ------------------------------------------------- |
| 알고리즘  | XGBClassifier                                     |
| 피처 수   | 50개                                              |
| 적용 대상 | `is_valid_click10 == 1` AND `click_cnt >= 30` |
| 아티팩트  | `models/model1_artifacts/`                      |
| 성능      | Test AUC ~0.85                                    |

### 7.3 피처 구성

| 출처        | 피처                                                                                                                                                                                  |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 광고 속성   | `ads_reward_price`, `ads_rejoin_type`, `ads_order`, `ads_action_rule`, `ads_action_diff_flag`, `ads_day_cap`, `analysis_ads_type_label`, `reward_band`                |
| 광고 마스터 | `reg_hour`, `reg_weekday_enc`, `reg_is_weekend`, `reg_hour_band`, `ads_require_adid`, `ads_os_type`, `action_target_cnt`, `mentioned_media_cnt`, `target_media_cnt` |
| 분류        | `final_media`, `final_action`                                                                                                                                                     |
| 변환        | OHE (카테고리형), Label Encoding (`ads_rejoin_type`, `ads_action_rule`)                                                                                                           |

### 7.4 스코어링 파이프라인

```
raw data
  → OHE 적용 (pipeline['ohe'])
  → Label Encoding (art['le_dict'])
  → 수치형 + OHE 결합
  → clean_col_names_unique() → align_features()
  → model.predict_proba(X)[:, 1]    ← raw probability
  → searchsorted(ref_proba_sorted, proba) / len(ref) × 100    ← 0~100 점수
  → pd.cut(score, bins, labels)     ← S/A/B/C/D 등급
```

### 7.5 등급 체계 (5-grade quantile cut)

| 등급        | 라벨     | 점수 범위    | 색상        | 운영 의미                         |
| ----------- | -------- | ------------ | ----------- | --------------------------------- |
| **S** | 최우수   | > 80.7       | `#1B5E20` | 적극 집행, 매체 확장 검토         |
| **A** | 우수     | 59.9 ~ 80.7  | `#2E7D32` | 안정 운영, S 진입 가능성 모니터링 |
| **B** | 보통     | 40.0 ~ 59.9  | `#F9A825` | 관찰, 소재/타겟 최적화 여지       |
| **C** | 주의     | 20.02 ~ 40.0 | `#E65100` | 경고, 개선 미시행 시 축소 검토    |
| **D** | 개선필요 | < 20.02      | `#C62828` | 즉시 축소/중단 검토               |

### 7.6 운영 상태 연동

M1 등급은 P1 오버뷰의 **운영 상태 배너**와 직결된다:

| M1 등급 | 운영 상태 |
| ------- | --------- |
| S, A, B | 정상      |
| C       | 주의      |
| D       | 긴급      |

## 8. M2: 조기부진 예측

### 8.1 목적

광고 집행 후 D+0 ~ D+3 초기 실적을 기반으로, 향후 부진할 위험을 예측한다. 부진 광고를 1~2주 사후에 발견하는 대신 D+3 시점에 조기 경보를 발령하여 선제 대응을 가능하게 한다.

### 8.2 모델 사양

| 항목      | 내용                         |
| --------- | ---------------------------- |
| 알고리즘  | LightGBM                     |
| 피처 수   | 101개                        |
| 적용 대상 | `is_valid_click10 == 1`    |
| 아티팩트  | `models/model2_artifacts/` |
| 성능      | Test PR-AUC ~0.72            |

### 8.3 피처 구성

M1 공통 피처 50개 + TF-IDF (`ads_name` + `ads_summary` 텍스트) + `early_click` (D+3 이내 클릭 합계)

### 8.4 의사결정 로직

```
early_click < 10 (min_early_click)
  → "rule_based_review"  (데이터 불충분, 룰 기반 검토)

early_click >= 10:
  m2_proba >= threshold → "decline_risk"   (부진 위험)
  m2_proba <  threshold → "normal"         (정상)
```

### 8.5 Threshold 옵션

| 키                 | 설명                                 | 기본값 |
| ------------------ | ------------------------------------ | ------ |
| `best_f1`        | F1 점수 최적화 threshold (기본 사용) | ~0.54  |
| `default_05`     | 기본 0.5                             | 0.50   |
| `recall_targets` | 타겟 재현율별 threshold              | dict   |

**Threshold 재적용**: `reapply_threshold(scores_df, threshold_value)` — 캐시된 `m2_proba`에 새 threshold를 적용하여 `m2_decision`을 재계산한다. 모델 재추론 불필요.

## 9. 배지 시스템

### 9.1 개요

ML 모델 결과를 운영자가 즉시 이해하고 행동할 수 있도록, **배지(Badge)** 기반 액션 제시 체계를 운영한다. 배지는 P1~P4의 ML 인사이트 섹션에서 표시되며, 클릭 시 P5 광고 상세로 진입한다.

### 9.2 기회 배지 (M1 S/A 등급 광고)

`_assign_opportunity_badge()` — 우선순위 순서:

| 우선순위 | 배지               | 조건                        | 아이콘 | 색상        | 운영 액션                                |
| -------- | ------------------ | --------------------------- | ------ | ----------- | ---------------------------------------- |
| 1        | **등급경계** | A등급 AND`m1_score >= 75` | ◐     | `#6A1B9A` | S등급 진입 직전 — 소재 최적화, 모니터링 |
| 2        | **매체확장** | S등급 또는 나머지 A등급     | ↗     | `#1565C0` | 다른 매체로 확장 가능성 검토             |

### 9.3 위험 배지 (M2 decline_risk 광고)

`_assign_risk_badge()`:

| 배지               | 조건                 | 아이콘 | 색상        | 운영 액션                            |
| ------------------ | -------------------- | ------ | ----------- | ------------------------------------ |
| **즉시조치** | `m2_proba >= 0.70` | ⚠     | `#C62828` | 소재 교체, 랜딩 점검, 일시 중단 검토 |
| **우선검토** | `m2_proba < 0.70`  | 👁     | `#E65100` | 3일 재평가, 핵심 지표 모니터링       |

### 9.4 배지 → P5 진입 흐름

ML 인사이트에서 "상세" 버튼 클릭 시:

```python
st.session_state["detail_ads_idx"]  = ads_idx       # 광고 ID
st.session_state["detail_badge"]    = badge          # 매체확장/등급경계/즉시조치/우선검토
st.session_state["detail_model"]    = "m1" or "m2"   # 어느 모델에서 진입
st.session_state["detail_source"]   = source_page    # 돌아가기 대상 페이지
→ st.switch_page("pages/05_광고_상세.py")
```

| - | --------------------- | ----------------------------- | ---- | -------------------------------------------------------- |

### 11.3 운영 상태 배너

**상단 배너** (`render_status_banner()`):

- M1 등급 기반 운영 상태 분류 → 정상/주의/긴급 비율 계산
- 배경색: 정상비율 ≥80% 초록 / ≥60% 노랑 / <60% 빨강
- 표시: 정상 비율 + 긴급 N건 + 주의 N건 + ML 커버리지

**하단 배너** (`render_dependency_banner()`):

- TOP 10 전환수 의존도 = 상위 10개 전환수 합 / 전체 전환수 × 100
- ≥70% 빨강 / ≥55% 노랑 / <55% 초록
- Expander로 의존 광고 리스트 (ML 스코어 포함 테이블)

### 11.4 시간대 × 요일 히트맵 섹션

레이아웃: `columns([3, 2])`

**좌측**:

- 24시간 × 7요일 CVR 히트맵 (`make_heatmap()`)
- 색상 스케일: `#CCD6EE` → `#6B8AC4` → `#4A6BAA`
- 클릭수 30 미만 셀은 NaN (회색)
- 하단: 요일 pills (multi-select) + 시간대 pills (6시간 단위 분할)

**우측**:

- 기본: 전체 기간 CVR Top 10 테이블 (클릭 ≥30)

## 18. 운영 규칙 엔진

### 18.1 운영 상태 분류 (`classify_operation_status()`)

M1 등급 기반. 모델 미적용 광고는 집계에서 제외.

| 상태 | M1 등급 | 색상 |
| ---- | ------- | ---- |
| 긴급 | D       | 빨강 |
| 주의 | C       | 주황 |
| 정상 | S, A, B | 초록 |

배너 색상: `normal_pct` ≥ 80% → 초록, ≥ 60% → 노랑, < 60% → 빨강

### 18.2 TOP 10 의존도 (`calc_dependency_ratio()`)

```
의존도 = sum(전환수 상위 10개) / sum(전체 전환수) × 100
```

| 의존도 | 색상 | 기준                          |
| ------ | ---- | ----------------------------- |
| ≥ 70% | 빨강 | 분석 기간 일별 75%ile = 76.5% |
| ≥ 55% | 노랑 | 분석 기간 일별 중앙값 = 60.4% |
| < 55%  | 초록 | 분석 기간 일별 25%ile = 50.7% |

### 18.3 사분면 분류 (`classify_quadrant()`)

X축 지표와 Y축 지표의 평균을 기준으로 4사분면:

```
          Y축 (마진율) 높음
              │
   소재 개선  │  최우수
   (blue)     │  (green)
──────────────┼────────────── X축 (효율) 좋음
   비효율     │  단가 점검
   (red)      │  (yellow)
              │
          Y축 낮음
```

- `is_lower_better`: CPA → 낮을수록 좋음 (X축 방향 반전)

### 18.4 CVR·마진율 알람 이모지

| 조건                         | 이모지 | 라벨             |
| ---------------------------- | ------ | ---------------- |
| clk < 30 또는 turn < 1       | ⊘     | 판단 보류        |
| CVR ≥ avg AND 마진율 ≥ avg | ★     | 최우수           |
| CVR < avg AND 마진율 ≥ avg  | ↺     | 소재 개선 후보   |
| CVR ≥ avg AND 마진율 < avg  | ⚠     | 수익율 점검 필요 |
| CVR < avg AND 마진율 < avg   | —     | 표시 안함        |

### 18.5 룰 기반 운영 알림 (`generate_alerts()`)

| 조건                                    | 레벨   | 메시지                               |
| --------------------------------------- | ------ | ------------------------------------ |
| `margin_rate < 0`                     | red    | "주의: {name} — 비용 초과"          |
| `cvr < avg_cvr × 0.7`                | yellow | "관찰: {name} — CVR 평균 대비 낮음" |
| `cvr >= avg_cvr AND margin_rate > 10` | green  | "양호: {name} — 안정"               |

정렬: red → yellow → green
